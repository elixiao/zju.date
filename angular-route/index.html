路由参数

关于参数的快照（snapshot）和参数的订阅（subscribe）

那参数快照和参数订阅有什么区别呢？应该在什么场景下使用呢？

  • 当路由参数变化不需要销毁旧组件，创建新组件时，即从 DOM 中移除当前的组件的实例，并用新参数重新创建，那可能导致界面抖动。更好的方式是复用同一个组件实例，并更新这些参数。如果有可能连续多次导航到此组件，那么就该用参数订阅（subscribe）或 paramMap 可观察对象的方式。
  • 如果用户总是会先返回列表页，再进入详情，不存在从一个详情页导航到另一个详情页，而不用经过列表的情况，即每次都会创建一个全新的组件实例。这个时候，建议使用参数快照（snapshot）的方式。

路由重定向

重定向路由需要一个pathMatch属性，来告诉路由器如何使用URL去匹配路由的路径，否则路由就会报错，因此要把pathMacth设置为’full’

Wild Cards（通配符）

注意：配置路由，一定要记得引入文件

每个 Route 都会把一个 URL 的 path 映射到一个组件，而通配符路由的 path 是两个星号（’**’），它会匹配任何 URL。 当路由器匹配不上以前定义的那些路由时，它就会选择这个路由。 通配符路由可以导航到自定义的 404 Not Found 组件，也可以重定向到一个现有路由。路由器使用先匹配者优先的策略来匹配路由，所以，具体路由应该放在通用路由的前面。
