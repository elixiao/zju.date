在my-customer.html里面这样写：

Name: {{customerInfo.name}} Address: {{customerInfo.address}}

那么：

就会得到不同的显示：

姓名: 刘德华，地址: 中山路29号
姓名: 周瑞发，地址: 南京路43号

这里面关键在于这句话：

给指令自己创建了一个scope，里面有个变量叫做customerInfo，这个scope可以接收来自controller里面的变量，把controller里面的info变量赋值给指令scope里面的customerInfo。嗯哼，看起来不错~

scope可以取三个值：false, true, object.

  1. Scope = false

    使用父作用域，操作的是同一个变量。

  2. Scope = true

    创建新的作用域，可以访问父作用域，但如果有同名本地变量则使用本地变量。

  3. Scope = {}

    创建一个完全独立的作用域。那如果想访问父作用域变量咋办？别急，有办法：

    1. @：获取父作用域只读权限（单向绑定）
    2. =：获取父作用域读写权限（双向绑定）

    3. &：调用父作用域函数

招式六：操作DOM的link函数

招式说明：降龙十八掌！

link函数的用法：

function link(scope, element, attrs, controller, transcludeFn) { ... }

其中：

  • scope是作用域
  • element是jqlite包装后的dom元素
  • attrs是该元素的一些属性
  • controller是require属性声明的控制器.
  • transcludeFn不知道！
