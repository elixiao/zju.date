YAML学习笔记

列表

YAML格式：

- Mark McGwire
- Sammy Sosa
- Ken Griffey

JSON格式：

["Mark McGwire", "Sammy Sosa", "Ken Griffey"]

总结：列表就是数组。

字典

YAML格式：

hr:  65
avg: 0.278
rbi: 147

JSON格式：

{
    "hr": 65,
    "avg": 0.278,
    "rbi": 147
}

总结：列表在中括号[]内，字典在大括号{}内。

字典和列表混合

1. 嵌套列表

YAML格式：

- - name
  - hr
  - avg
- - Mark McGwire
  - 65
  - 0.278
- - Sammy Sosa
  - 63
  - 0.288

JSON格式：

[
  [
    "name",
    "hr",
    "avg"
  ],
  [
    "Mark McGwire",
    65,
    0.278
  ],
  [
    "Sammy Sosa",
    63,
    0.288
  ]
]

总结：嵌套列表就是把列表作为元素放到新列表里面。

2. 嵌套字典

YAML格式：

Mark McGwire: 
  hr: 65
  avg: 0.278
Sammy Sosa: 
  hr: 63
  avg: 0.288

JSON格式：

{
  "Mark McGwire": {
    "hr": 65,
    "avg": 0.278
  },
  "Sammy Sosa": {
    "hr": 63,
    "avg": 0.288
  }
}

总结：嵌套字典就是把字典作为另一个字典的value值。

3. 列表里面放字典

YAML格式：

- name: Mark McGwire
  hr:   65
  avg:  0.278
- name: Sammy Sosa
  hr:   63
  avg:  0.288

JSON格式：

[
    {
        "name": "Mark McGwire",
        "hr": 65,
        "avg": 0.278
    },
    {
        "name": "Sammy Sosa",
        "hr": 63,
        "avg": 0.288
    }
]

4. 字典里面放列表

YAML格式：

american:
  - Boston Red Sox
  - Detroit Tigers
  - New York Yankees
national:
  - New York Mets
  - Chicago Cubs
  - Atlanta Braves

JSON格式：

{
  "american": [
    "Boston Red Sox",
    "Detroit Tigers",
    "New York Yankees"
  ],
  "national": [
    "New York Mets",
    "Chicago Cubs",
    "Atlanta Braves"
  ]
}

分隔文档

YAML格式：

---
- Mark McGwire
- Sammy Sosa
- Ken Griffey
---
time: 20:03:47
player: Sammy Sosa
action: grand slam

JSON格式：

[
  [
    "Mark McGwire",
    "Sammy Sosa",
    "Ken Griffey"
  ],
  {
    "time": 72227,
    "player": "Sammy Sosa",
    "action": "grand slam"
  }
]

总结：多个YAML文档可以放在一个文件中，使用—分隔开（一个YAML文档可理解为类似JSON的一个对象）

锚点

YAML格式：

anchored_content: &anchor_name your string
other_anchor: *anchor_name

base: &base
  name: your name
  sex: your sex
foo: 
  <<: *base
  age: 10
bar: 
  <<: *base
  sex: male

JSON格式：

{
  "anchored_content": "your string",
  "other_anchor": "your string",
  "base": {
    "name": "your name",
    "sex": "your sex"
  },
  "foo": {
    "name": "your name",
    "sex": "your sex",
    "age": 10
  },
  "bar": {
    "name": "your name",
    "sex": "male"
  }
}

总结：用&定义一个引用标签，用*引用这个标签的数据，用<<进行hash值合并。

文本块

YAML格式：

literal_block: |
  This entire block of text will be the value of the 'literal_block' key,
  with line breaks being preserved.

  The literal continues until de-dented, and the leading indentation is
  stripped.

    Any lines that are 'more-indented' keep the rest of their indentation -
    hese lines will be indented by 4 spaces.


folded_style: >
  This entire block of text will be the value of 'folded_style', 
  but this time, all newlines will be replaced with a single space.

  Blank lines, like above, are converted to a newline character.

    'More-indented' lines keep their newlines, too -
    this text will appear over two lines.

JSON格式：

{
  "literal_block": "This entire block of text will be the value of the 'literal_block' key,\nwith line breaks being preserved.\n\nThe literal continues until de-dented, and the leading indentation is\nstripped.\n\n  Any lines that are 'more-indented' keep the rest of their indentation -\n  hese lines will be indented by 4 spaces.\n",
  "folded_style": "This entire block of text will be the value of 'folded_style',  but this time, all newlines will be replaced with a single space.\nBlank lines, like above, are converted to a newline character.\n\n  'More-indented' lines keep their newlines, too -\n  this text will appear over two lines.\n"
}

总结：literal_block完整保留所有内容，包括换行；folded_style:不转换普通换行，只把空行和缩进当做换行。

复杂键值

YAML格式：

? - Detroit Tigers
  - Chicago cubs
:
  - 2001-07-23
  - 2011-08-01

? |
this is block
new line
: [ item1, item2, item3 ]

JSON格式：

{
  "Detroit Tigers,Chicago cubs": [
    "2001-07-23T00:00:00.000Z",
    "2011-08-01T00:00:00.000Z"
  ],
  "this is block\nnew line\n": [
    "item1",
    "item2",
    "item3"
  ]
}
