package.json中包的版本语法

两种通配符：

  • ^ 大版本（major）
  • ~ 小版本（minor）

会安装大于或等于 0.5.0 的任意 0.x.x 版本，但不会安装 1.x.x 版本。能够保证在 0.x.x 版本中保持最新。

会安装任意 0.5.x 版本，但不会安装 0.6.x 版本，能够保证在 0.5.x 中保持最新

这是一种标准的版本格式语义规则：

版本格式：主版本号.次版本号.修订号，版本号递增规则如下：

  • 主版本号：当你做了不兼容的API 修改，
  • 次版本号：当你做了向下兼容的功能性新增，
  • 修订号：当你做了向下兼容的问题修正。

先行版本号及版本编译信息可以加到“主版本号.次版本号.修订号”的后面，作为延伸。
