docker 目录挂载

Volume 在 docker 中就是存在于宿主机上目录或者文件，它可以将容器以及容器产生的数据分离开来，当用户使用 docker rm container 删除容器时，不会影响相关的数据。

Volume可以使用以下两种方式创建：

  • 在 Dockerfile 中指定 VOLUME /some/dir
  • 执行 docker run -v /some/dir 命令来指定

Docker 会在宿主机上 /var/lib/docker 目录的下面创建一个新的目录，然后将其挂载到容器内指定的路径中，如果在容器中不存在指定的路径，那么该目录将会被自动创建。当删除使用该 Volume 的容器时，Volume 本身不会受到影响，它可以一直存在下去。不过也可以同时删除容器和 Volume：
