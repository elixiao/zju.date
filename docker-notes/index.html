docker 笔记

基础

  1. 检查 Docker 是否启动：docker info
  2. help 选项：docker help 和 docker <command> —help

镜像

  1. 搜索仓库：docker search <image>

  2. 查看镜像：docker images

  3. 下载镜像：docker pull <image>

    如果你使用的是 Docker for Mac 版本，那么所有的docker images 保存在下面这个文件里：

  4. 获取镜像历史：docker history <image>

    注：只能对本地存在的 Docker 镜像执行这个命令。例如： docker history busybox

  5. 删除镜像：docker rmi <image>

    如果该镜像创建的 container 正在运行，则不能删除这个镜像。另外，可能会有很多层的依赖，所以有时候需要用 -f 参数进行删除。

  6. 推送镜像： docker push <user>/<repo_name>

    注：不能直接使用 docker push repo_name，所有推送命令的格式必须是 docker push <user>/<repo_name>，不加 <user> 无法推送到 Docker registry。

容器

  1. 显示容器：docker ps 命令用于显示运行中的容器

    docker ps -a 则是显示所有容器，包括已停止的容器。

  2. 启动容器：docker run [OPTIONS] <image> [COMMAND][ARG…]

    -d : 以后台进行方式运行容器

    -t : 提供一个伪终端

    -i : 提供交互输入，一般与“-t”一起使用，如果只提供“-i”选项，则容器启动后是无法退出的

    -v : 映射一个volume给容器，如： -p /data/www:/var/www/html

    -p : 将容器的端口映射给宿主机，如： -p 8080:80

    例如：

    • 启动容器并执行echo命令：docker run <image> /bin/echo Hello Docker

      这条命令是运行容器中的 /bin/echo 命令，参数是 Hello Docker

    • 交互模式：docker run -t -i <image> /bin/bash

      -t 选项让 docker 分配一个伪终端并绑定到容器的标准输入上，-i 让容器的标准输入保持打开。

    • 运行守护进程：docker run -d -p 8080:80 --name webserver nginx ，运行 nginx 镜像实例，-d 或者 --detach 代表后台运行，-p 代表绑定端口 8080 到 docker 的80

      守护进程：daemon通常是指在后台持续运行的服务，例如nginx、apache、php 等。Docker 的 daemon 模式便是类似 nginx 一样，让Docker 运行在后台提供服务。和大多数的 unix 服务一样，-d 参数可以让 Docker 运行在后台。

  3. 停止容器：docker stop <container>

  4. 重启容器：docker restart <container>

  5. 删除容器： docker rm <container>

    快速删除容器： docker ps -a 获取ID，不冲突的情况下，使用 docker rm 92 命令快速删除容器 920c0f3c240a 。Docker 的文件是放在 /var/lib/docker/containers/ 中的。

  6. 保存容器：docker commit <container> <image>

    注意：镜像名称中只允许出现小写字母和数字，这里保存成一个名叫 image_name 的镜像。

  7. 检查容器/镜像：docker inspect <container>/<image>，用于收集容器/镜像的底层信息，包括如下内容：

    • 实例的IP地址
    • 端口绑定
    • 端口映射（容器可以直接使用命令：docker port container）
    • 搜集配置信息

  8. 查询容器输出：docker logs <ID>/<name>

    例如：通过 docker logs 命令查询容器的输出内容。

  9. 进入容器：docker attach <container> ，进入容器并与运行中的容器进行交互，同时允许查看容器内守护进程态下的程序的输出，使用以下两种方式退出容器：

    1. Ctrl + C 直接退出
    2. Ctrl + \ 退出并显示堆栈信息。

    如果container当前在运行bash，CTRL-C 则是当前行的输入，没有退出；如果container当前正在前台运行进程，例如输出 nginx 的 access.log 日志，CTRL-C不仅会导致退出容器，而且还stop了。这不是我们想要的，detach的意思按理应该是脱离容器终端，但容器依然运行。好在attach是可以带上--sig-proxy=false来确保CTRL-D或CTRL-C不会关闭容器。

  10. 进入容器：docker exec

    docker attach 让用户可以进入 container 查看输出等等操作，但是并不会另外启动一个进程，如果你用 ctrl-c 来退出，同时这个信号会 kill container（默认情况）

    docker exec 会启动另外一个进程来进入 container，这里的操作是在这个进程下的。如果你用 ctrl-c 来退出，不会kill 原来的 container

  11. 显示改动：docker diff <contianer> 用于显示容器的文件改动，包括A（add），C（change），D（delete）。这些信息可以用于debug，也可以用于快速信息共享。

高级

  1. 获取事件：获取 Docker 服务端的实时事件，例如 docker events 的同时运行 docker run ubuntu。

  3. 导出文件：和 import 相反，export 将容器导出成 tar 压缩包。

  4. 复制文件：从容器内复制出文件：格式 docker cp container:path hostpath

  5. 登录服务器：

  6. 退出信号值：wait用来获取容器的退出信号值。已退出的容器直接返回 exit
    值，未退出的容器等待容器退出，然后返回 exit 值。

  7. 构建：用 Dockerfile 生成镜像文件。

    • PATH / URL / - 这三个是必选选项，三选一，- 选项非常有意思：docker build -，然后可以交互式地输入Dockerfile，输入完毕后按 Ctrl+D 结束输入即可。OPTION 一定要放在 PATH / URL / - 前面。
    • OPTION 可以使用 docker build --help 进行查看，主要包括构建过程中的内存限制、CPU限制、以及其他。有两个必须需要了解的 OPTION 选项：
      1. -f：和 make 一样，如果 PATH 内找不到 Dockerfile，则 docker build 必须要 -f 参数指定Dockerfile。
      2. -t： 指定生成的镜像标签。例如：docker build -f Dockerfile -t tag .，使用 Dockerfile.bak 生成一个标签叫 tag 的镜像文件。

  8. 加载和输出：使用 load 从 stdin 导入一个 tar 格式的镜像或者仓库，然后用 save 将 tar 镜像输出到 stdout。

    • docker save -o <output file> <image> 或者 docker save <image> > <output file>
    • docker load -i <file name> 或者 docker load < <file name>
