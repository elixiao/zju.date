在 render 中更新 props 和 state

大体意思就是在 render 这种需要 props 和 state 进行渲染的方法中，不能再对 props 和 state 进行更新。因为React 会在 props 和 state 改变的时候调用 render 进行 DOM diff 然后渲染，如果在渲染过程中再对 props 和 state 进行更改，就陷入死循环了。onPress 应该传递进一个函数，这个函数会在点击的时候执行。一种可行的写法是使用匿名函数封装：

这样就不会在渲染的时候跑里面的函数，也就避免了冲突。

TouchableHighlight 只能有一个子元素

TouchableHighlight只能有一个子元素，如果需要在其内部放置多个组件，需要先把组件都放在一个View里面，然后再把这个View放在TouchableHighlight里面才行。

scrollView 和 listView 无法混用

父子组件通信

父组件通过props把数据传递给子组件，子组件传递数据给父组件是通过触发事件，委托调用父组件的处理函数，从而实现把值传给父组件

react 和 react native 代码无法共用

React 和 React Native 的原理是相同的，都是由 js 实现的虚拟 dom 来驱动界面 view 层渲染。但 React 是驱动 html dom 渲染； React Native 是驱动 android/ios 原生组件渲染。

第二种情况：也就是其他情况，你感觉什么都是好的但是运行起来还是会报这个错误。那么很有可能是8081端口被占用了。你可以尝试：切换到项目所在目录，输入react-native start 如果出现Packager can't listen on port 8081那说明端口被占用了。根据命令行提示进行操作：

  1. 找到被占用的端口：lsof -n -i4TCP:8081
  2. 杀掉该进程：kill -9 <PID>
  3. 重新运行：react-native run-ios/android

NDK is missing a “platforms” directory.

去 https://developer.android.google.cn/ndk/downloads/ 下载 ndk，解压之后放到指定的目录下即可。

处理方法是把module下的signingConfigs 注释掉即可

原因:AndroidMainfest.xml的配置不对导致找不到MainActivity这个类

  1. 首先是检查这个MainActivity.java是不是真的存在，且包名和路径无误；
  2. 如果文件存在，且包名和路径没有问题，那么就打开你项目所在 的/android/app/build.gradle文件，删除defaultConfig下的applicationId，或者将applicationId的值改成manifest文件里指定的package。
  3. 再运行”react-native run-android”，OK了。

出现这个问题的原因是你的package和applicationId不一致。如果你没有在你的 build.gradle 文件中定义 applicationId，这个applicationId 将默认为 AndroidManifest.xml 中所指定的与package相同的值。
注意：现在的Android Gradle编译系统的android项目是以applicationId作为唯一标识应用的包名。

说明 RCTSplashScreenModule 在 MainApplication.java 中被 import 了两次或者生成了两个实例，类似其他的也一样。
