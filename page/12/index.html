路由命令

添加路由规则：

  • ip route show：显示路由表；
  • ip route add：添加路由；
  • ip route change：修改路由；
  • ip route replace：修改路由或添加路由；
  • ip route delete：删除路由；
  • ip route get：获得单条路由的详细信息；
  • ip route flush：清空路由表；

Route 命令是用于操作 ip 路由表，它创建一个静态路由让指定主机或者网络通过某个网络接口，如 eth0。

  • must 代表且（AND）
  • must_not 代表非（NOT）

  • match_phrase：匹配一个短语，例如查询【中国人】，被分词为【中国】和【人】，先在指定字段查找到 “中国”，然后在”中国”后面去查找 “人”，对顺序有要求
  • _delete_by_query：删除所有匹配的文档

docker中的IP网段

一般情况下，容器中可以直接访问的宿主机暴露的端口，今天换了一台机器却发现 docker 容器不能访问宿主机了，经过查阅资料，最终找到了原因，涉及到一些基础概念，这里记录如下：

IP 网段

IP地址分为五类:

  • A类保留给政府或大型企业
  • B类分配给中等规模的公司
  • C类分配给小公司或个人
  • D类用于组播
  • E类用于实验

其中最常见的就是前三类A、B和C，当将 IP 地址写成二进制形式时，他们有如下特征：

  1. A 类地址

    • 第一位总是 0，如 10.0.0.1==00001010-00000000-00000000-00000001
    • 范围：1.0.0.0-127.255.255.255
    • 默认子网掩码为 255.0.0.0
    • 最多可容纳 16777215 台计算机

  2. B 类地址

    • 前两位总是 10，如 172.16.0.1==10101100-00010000-00000000-00000001
    • 范围：128.0.0.0-191.255.255.255
    • 默认子网掩码为 255.255.0.0
    • 最多可容纳 65535 台计算机

  3. C 类地址

    • 前三位总是 110，如 192.168.0.1==11000000-10101000-00000000-00000001
    • 范围：192.0.0.0-223.255.255.255
    • 默认子网掩码是 255.255.255.0
    • 最多可容纳 254 台计算机
