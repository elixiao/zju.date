CSS white-space 用法

  • pre：保留空格，不自动换行，直到遇到换行符
  • pre-wrap：自动换行
  • pre-line：自动换行，并把多个空格合并成一个

normal：连续的空白符会被合并，换行符会被当作空白符来处理。填充line盒子时，必要的话会换行。
nowrap：和 normal 一样，连续的空白符会被合并。但文本内的换行无效。
pre：连续的空白符会被保留。在遇到换行符或者 <br> 元素时才会换行。
pre-wrap：连续的空白符会被保留。在遇到换行符或者 <br> 元素，或者需要为了填充line盒子时才会换行。
pre-line：连续的空白符会被合并。在遇到换行符或者 <br> 元素，或者需要为了填充line盒子时会换行。

换行符 空格和制表符 文字转行
normal 合并 合并 转行
nowrap 合并 合并 不转行
pre 保留 保留 不转行
pre-wrap 保留 保留 转行
pre-line 保留 合并 转行

Nginx 中的 location

语法：

修饰符有四种：

  1. = ：精确匹配并终止搜索
  2. ~：区分大小写的正则表达式匹配
  3. ~*：不区分大小写的正则表达式匹配
  4. ^~：字符串打头匹配（若是最佳匹配，则不进行正则表达式匹配）

优先级顺序：

  1. 精确匹配(=)
  2. 字符串打头匹配(^~)
  3. 正则匹配(~ 或 ~*)
  4. 通用匹配(/)

两种正则当中，区分大小写的优先级高，也就是不带 * 的优先级高
