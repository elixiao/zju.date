git 分支管理

Git 分支管理

首先要明确下面两个分支的概念：

分支 说明
develop 包含的是下个版本需要发布的内容
master 包含的是可以部署到生产环境中的代码

意思是说：

  • develop 分支，是用于随时可以发布到 develop 的代码，便于线上测试；
  • master 分支，是用于随时可以发布到 master.com 生产环境的代码。
