CSS伪元素

伪元素(Pseudo Element)，顾名思义“不是网页里面真正的元素”，但看起来和真正元素一样，是它必须依附在一个真实的元素上。

在 W3C 的定义里总共有五个伪元素 ( 其他仍在测试阶段 )，分别是

  • ::before
  • ::after
  • ::first-line
  • ::first-letter
  • ::selection

::before和::after

::before 是在原本的元素“之前”加入内容，::after 则是在原本的元素“之后”加入内容，两者的表现形式都是 display:inline-block。

将分支 A 上的某些 commit 提交到分支 B

cherry-pick 直译为捡樱桃，在 git 里，它的功能是把已经存在的 commit 进行挑选，然后重新提交。比较适合的一个场景是把 A 分支某次提交的功能也提交到 B 分支上，使用方法:

  1. 先切换到 A 分支

  2. 找到这次提交

  3. 再切换到 B 分支

  4. 把 A 分支下的这次 commit 重新提交到 B 分支下

当执行完 cherry-pick 以后，将会生成一个新的提交，这个新的提交的哈希值和原来的不同，但标识名一样。
