路由命令

添加路由规则：

  • ip route show：显示路由表；
  • ip route add：添加路由；
  • ip route change：修改路由；
  • ip route replace：修改路由或添加路由；
  • ip route delete：删除路由；
  • ip route get：获得单条路由的详细信息；
  • ip route flush：清空路由表；

Route 命令是用于操作 ip 路由表，它创建一个静态路由让指定主机或者网络通过某个网络接口，如 eth0。

ES 中可以用 bool 查询组合出各种复杂的查询条件，在使用 bool 查询的时候，只需要记住下面三点即可：

  • should 代表或（OR）
  • must 代表且（AND）
  • must_not 代表非（NOT）

最后一个 bool 查询语句包含了一个 match_all 查询（查询部分），和一个 range（范围）查询（过滤部分）。我们可以将查询和过滤部分替换成任何其它的查询。

除了用 bool 组合基本关键词之外，ES 也提供了其他更为便捷的关键词，例如：

  • multi_match：匹配多个字段，只要其中一个字段包含分词后的文档就能被搜索到
  • match_phrase：匹配一个短语，例如查询【中国人】，被分词为【中国】和【人】，先在指定字段查找到 “中国”，然后在”中国”后面去查找 “人”，对顺序有要求
  • _delete_by_query：删除所有匹配的文档

flex-grow 和 flex-shrink

flex-grow 属性的作用：

父元素在主轴方向还有剩余空间时，子元素们如何分配这些剩余空间。

其值为一个权重（也称扩张因子），默认为 0（纯数值，无单位），剩余空间将会按照这个权重来分配。

比如剩余空间为 x，三个元素的 flex-grow 分别为 a，b，c，求和 sum = a + b + c。那么三个元素权重是 a/sum, b/sum, c/sum，就是按照这个比例分配剩余空间的。

举个例子：

父元素宽度 500px，三个子元素的 width 分别为 100px，150px，100px，于是剩余空间为 150px。

如果三个元素的 flex-grow 分别是 1，2，3，那么 sum 为 6，则三个元素被分到的空间是：

  • 150 * 1 / 6 = 25px
  • 150 * 2 / 6 = 50px
  • 150 * 3 / 6 = 75px

三个元素最终的宽度分别：

  • 100px + 25px = 125px
  • 150px + 50px = 200px
  • 100px + 75px = 175px
