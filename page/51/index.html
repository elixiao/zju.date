git push -u

git push 的 -u 参数具体适合含义？

一般只有同时存在多个远程仓库时才会用到 –set-upstream 。每个 git branch 可以有个对应的 upstream。假设你有两个upstream，分别叫 server1 和 server2，本地 master branch 的 upstream 是 server1 上的 master，那么当你不带参数直接输入 git pull 或者 git push 时，默认是对 server1 进行 pull/push。如果你成功运行 git push -u server2 master，那么除了本地 branch 会被 push 到 server2 之外，还会把 server2 设置成 upstream。

上面命令表示，将当前分支推送到origin主机的对应分支，如果当前分支只有一个追踪分支，那么主机名都可以省略。

CSS几个属性

任何一个div，都有margin、border、padding，位置如下：

那下面几个重要的属性都是如何计算的呢？

  • width/height：div内部不含padding，仅用于展示的那块区域的宽度和高度。

  • clientWith/clientHeight：div内部包含padding不含border的宽度和高度。

  • offsetWidth/offsetHeight：div外部不含margin包含border的宽度和高度。

  • scrollWidth/scrollHeight：可滑动区域的宽度和高度

  • offsetLeft/offsetTop：div左上角（不含margin）距离最近positioned元素的宽度和高度。

  • clientLeft/clientTop：左边框和上边框的宽度和高度。

  • scrollLeft/scrollTop：超出部分的宽度和高度。

赌金分配问题

A和B打赌，各出x元，N局决胜负，赢者获取全部赌金。然而当A赢a局，B赢b局时赌局不得不终止，请问赌金如何分配？

读完题目，分析本题的应用场景，从基本常识上来讲，我觉得有3个非常明显的结论：

  1. 如果a=b<N，那么赌金平分，即A、B各获得x元。
  2. 如果a<b<N，A拿回的赌金必然小于B拿回的赌金。
  3. A赢a局，B赢b局时A拿到的赌金等于A赢b局，B赢a局时B拿到的赌金。
