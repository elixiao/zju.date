docker 笔记

基础

  1. 检查 Docker 是否启动：docker info
  2. help 选项：docker help 和 docker <command> —help

镜像

  1. 搜索仓库：docker search <image>

  2. 查看镜像：docker images

  3. 下载镜像：docker pull <image>

    如果你使用的是 Docker for Mac 版本，那么所有的docker images 保存在下面这个文件里：

  4. 获取镜像历史：docker history <image>

    注：只能对本地存在的 Docker 镜像执行这个命令。例如： docker history busybox

  5. 删除镜像：docker rmi <image>

    如果该镜像创建的 container 正在运行，则不能删除这个镜像。另外，可能会有很多层的依赖，所以有时候需要用 -f 参数进行删除。

  6. 推送镜像： docker push <user>/<repo_name>

    注：不能直接使用 docker push repo_name，所有推送命令的格式必须是 docker push <user>/<repo_name>，不加 <user> 无法推送到 Docker registry。

URL中的保留字

URL中有很多保留字，但是保留字出现在路径、查询或段的部分时，往往很难完全梳理清楚，例如 + 在路径部分是被允许的并且特指正号而不是空格。其他类似的有：

  • ? 在查询部分允许不被转义
  • / 在查询部分允许不被转义
  • = 在作为路径参数或者查询参数值以及在路径部分允许不被转义
  • :@-._~!$&'()*+,;= 等字符在路径部分允许不被转义
  • /?:@-._~!$&'()*+,;= 等字符在任何段中允许不被转义

这样下面的地址虽然看起来有点混乱：

http://example.com/:@-._~!$&'()*+,=;:@-._~!$&'()*+,=:@-._~!$&'()*+,==?/?:@-._~!$'()*+,;=/?:@-._~!$'()*+,;==#/?:@-._~!$&'()*+,;=

按照上面的规则，其实上是一个合法的地址，上面路径可以被解析为：

部分 值
协议 http
主机 example.com
路径 /:@-._~!$&'()*+,=
路径参数名 :@-._~!$&'()*+,
路径参数值 :@-._~!$&'()*+,==
查询参数名 /?:@-._~!$'()* ,;
查询参数值 /?:@-._~!$'()* ,;==
段 /?:@-._~!$&'()*+,;=
