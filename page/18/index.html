encodeURI 和 encodeURIComponent

这两个函数都是用于对 URL 进行编码的，那为什么要对 URL 进行编码？通常如果一样东西需要编码，说明这样东西并不适合传输。原因多种多样，如体积过大、包含隐私数据等，而对于 URL 来说，之所以要进行编码，是因为 URL 中有些字符会引起歧义。 举两个典型的例子：

  1. URL 参数字符串中使用 key=value 键值对这样的形式来传参，键值对之间以 & 符号分隔，如 /s?q=abc&ie=utf-8。如果你的value字符串中包含了 = 或者 &，那么势必会造成接收 URL 的服务器解析错误，因此必须将引起歧义的 & 和 = 符号进行转义，也就是对其进行编码。
  2. URL 的编码格式采用的是 ASCII 码，而不是 Unicode，这也就是说不能在 URL 中包含任何非 ASCII 字符，例如中文、日文、韩文等。就需要用 ASCII 码来对 Unicode 进行编码。

URL 编码的原则就是使用安全的字符（没有特殊用途或者特殊意义的可打印字符）去表示那些不安全的字符。

将分支 A 上的某些 commit 提交到分支 B

cherry-pick 直译为捡樱桃，在 git 里，它的功能是把已经存在的 commit 进行挑选，然后重新提交。比较适合的一个场景是把 A 分支某次提交的功能也提交到 B 分支上，使用方法:

  1. 先切换到 A 分支

  2. 找到这次提交

  3. 再切换到 B 分支

  4. 把 A 分支下的这次 commit 重新提交到 B 分支下

当执行完 cherry-pick 以后，将会生成一个新的提交，这个新的提交的哈希值和原来的不同，但标识名一样。
