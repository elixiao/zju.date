npm link

npm link 用来在本地项目和本地 npm 模块之间建立连接，可以在本地进行模块测试。如果是命令行工具的话，可以将一个任意位置的 npm包 链接到全局执行环境，从而在任意位置使用命令行都可以直接运行该 npm 包。

当运行 npm link 时发生了两件事：

  • 为 npm 包目录创建软链接，将其链到 {prefix}/lib/node_modules/<package>
  • 为可执行文件(bin)创建软链接，将其链到 {prefix}/bin/{name}

在 windows 中：

  • 目录: C:\Users\{Username}\AppData\Roaming\npm\node_modules\<package>
  • 文件: C:\Users\{Username}\AppData\Roaming\npm\<name>

在 Angular 中使用 debounce 和 throttle

debounce 防抖

  1. 名词释义

    对于某事件流，自触发时刻开始，若持续指定时间内未有事件发生，则触发该事件，否则更新该事件，直到满足持续指定时间未有事件发生的条件后再触发最后更新的这个事件。

    防抖的原理就是：你尽管触发事件，但是我一定在事件触发 n 秒后才执行，如果你在一个事件触发的 n 秒内又触发了这个事件，那我就以新的事件的时间为准，n 秒后才执行，总之，就是要等你触发完事件 n 秒内不再触发事件，我才执行!

  2. 应用场景

    搜索手牌或商品的时候，keyup事件并不需要每次都触发请求，而是当用户某两次keyup中间的间隔超过指定时间才去请求，否则用户噼里啪啦打字，你发了几十个请求，浪费资源。

git 分支管理

Git 分支管理

首先要明确下面两个分支的概念：

分支 说明
develop 包含的是下个版本需要发布的内容
master 包含的是可以部署到生产环境中的代码

意思是说：

  • develop 分支，是用于随时可以发布到 develop 的代码，便于线上测试；
  • master 分支，是用于随时可以发布到 master.com 生产环境的代码。
