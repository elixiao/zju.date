JavaScript 中的 jsonp 技术

如果两个页面拥有相同的协议（protocol）、端口（如果指定）和主机，那么这两个页面就属于同一个源（origin）。

下表给出了相对http://sub.domain.com/dir1/page.html同源检测的示例:

URL 结果 原因
http://sub.domain.com/dir2/other.html 成功
http://sub.domain.com/dir/inner/another.html 成功
https://sub.domain.com/secure.html 失败 协议不同
http://sub.domain.com:81/dir/etc.html 失败 端口不同
http://news.domain.com/dir/other.html 失败 主机名不同

同源策略是指基于安全考虑，当前域不能访问其他域的东西。可使用CORS来实现跨域访问。

所谓同源是指：域名、协议、端口相同的两个域才能访问数据。但是 <script> 的 src 可以避开同源策略，跨域获取数据。

给 <script> 的 src 属性添加一个地址，浏览器就会发送一个 http 请求下载 src 属性指向的 url，如果返回的来的格式是 text/javascript，那么就会立即执行。

因此，使用 script 元素进行 ajax 传输的两个主要原因是：

  1. 它不受同源策略的影响；
  2. 包含json编码的数据响应体会自动解码执行。

Linux命令之tee

ls > a.txt可以把输出保存到文件，但这时我们就不能看到输出了，如果我们既想把输出保存到文件中，又想在屏幕上看到输出内容，就要使用tee命令。

  1. tee

    只输出到标准输出。

  2. tee file

    保存到文件并输出到标准输出，如果文件不存在，则创建；如果已经存在，则覆盖。

  3. tee -a file

    输出到标准输出的同时，追加到文件file中。如果文件不存在，则创建；如果已经存在，就在末尾追加内容，而不是覆盖。

  4. tee -

    输出到标准输出两次，例如：

  5. tee file1 file2 -

    输出到标准输出两次，同时保存到file1和file2中。
