ElasticSearch学习笔记

ElasticSearch概述

与传统数据库的类比：

关系数据库      ⇒ 数据库(database) ⇒ 表(table)    ⇒ 行(row)         ⇒ 列(column)
Elasticsearch  ⇒ 索引(index)      ⇒ 类型(type)   ⇒ 文档(document)  ⇒ 字段(field)

一个 Elasticsearch 集群可以包含多个索引，每个索引可以包含1个类型，这个类型中包含多个文档，然后每个文档中又包含了很多的字段。但是在 6.x 的版本中 type 被废弃了，在后期的版本中被完全移除，因此可以把 ES 想象成一个数据库只能容纳一个表。废弃的原因文档里面说得也比较清楚，在传统数据库中，表跟表之前是相互独立的，可以拥有不同的字段、不同的索引，但是在 ES 中，同一个索引下不同类型里名称相同的字段会被 Lucene 视为同一个字段，这就会产生矛盾，例如两个类型里面都有 deleted 字段，想在其中一个类型定义它是日期格式，另一个类型中定义它是布尔值，ES 做不到这一点，但数据库是可以做到的。

JavaScript中的this

JavaScript 中的 this 有如下 5 种使用场景：

  1. 箭头函数 () => {}
  2. new 调用构造函数 new F()
  3. 显式绑定 call/apply/bind
  4. 隐式绑定 obj.foo()
  5. 默认绑定 foo()

箭头函数中的this

箭头函数没有自己的执行上下文，内部的 this 代表函数创建时候所在的词法作用域中的 this，与调用方式无关，例如：

由于是箭头函数，所以 this 就是根据词法作用域往上查找，找到了全局作用域，所以打印两个 undefined，而如果把箭头函数改成普通的 function，那么将会打印一个 david 一个 undefined。

总结：箭头函数的 this 与如何调用无关，只与创建时候的词法作用域有关。

JavaScript 中的 jsonp 技术

如果两个页面拥有相同的协议（protocol）、端口（如果指定）和主机，那么这两个页面就属于同一个源（origin）。

下表给出了相对http://sub.domain.com/dir1/page.html同源检测的示例:

URL 结果 原因
http://sub.domain.com/dir2/other.html 成功
http://sub.domain.com/dir/inner/another.html 成功
https://sub.domain.com/secure.html 失败 协议不同
http://sub.domain.com:81/dir/etc.html 失败 端口不同
http://news.domain.com/dir/other.html 失败 主机名不同

同源策略是指基于安全考虑，当前域不能访问其他域的东西。可使用CORS来实现跨域访问。

所谓同源是指：域名、协议、端口相同的两个域才能访问数据。但是 <script> 的 src 可以避开同源策略，跨域获取数据。

给 <script> 的 src 属性添加一个地址，浏览器就会发送一个 http 请求下载 src 属性指向的 url，如果返回的来的格式是 text/javascript，那么就会立即执行。

因此，使用 script 元素进行 ajax 传输的两个主要原因是：

  1. 它不受同源策略的影响；
  2. 包含json编码的数据响应体会自动解码执行。
