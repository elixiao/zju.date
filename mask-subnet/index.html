子网掩码计算

查看路由表：

IP 后面的数字 16 是用于计算子网网段的，其与子网掩码的映射关系表为：

/数字 IP总数 子网掩码 C段个数
/30 4 255.255.255.252 1/64
/29 8 255.255.255.248 1/32
/28 16 255.255.255.240 1/16
/27 32 255.255.255.224 1/8
/26 64 255.255.255.192 1/4
/24 256 255.255.255.0 1
/23 512 255.255.254.0 2
/22 1024 255.255.252.0 4
/21 2048 255.255.248.0 8
/20 4096 255.255.240.0 16
/19 8192 255.255.224.0 32
/18 16384 255.255.192.0 64
/17 32768 255.255.128.0 128
/16 65536 255.255.0.0 256

例如计算 10.10.15.10/16 的网段，先找到上表的 /16 表格，找到子网掩码 255.255.0.0，按照下面的步骤计算：

  1. 把 10.10.15.10 转换为二进制： 00001010.00001010.00001111.00001010

  2. 把 255.255.0.0 转换为二进制：11111111.11111111.00000000.00000000

  3. 把上面的结果进行与运算得到子网地址，然后再用子网地址与网络掩码取反进行异或运算，公式为：

    • 1 AND 1 = 1，1 XOR 1 = 0
    • 0 AND 1 = 0，0 XOR 1 = 1
    • 0 AND 0 = 0，0 XOR 0 = 0
    • 1 AND 0 = 0，1 XOR 0 = 1

  4. 把上述结果转换回十进制子网地址为：10.10.0.0，广播地址为：10.10.255.255

  5. 因此 10.10.0.0 到 10.10.255.255 之间的网段都是子网，其他的都不是。
