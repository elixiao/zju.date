JS 模块化

随着 ES6 的普及，Webpack、Parcel 等打包工具的出现，模块化 JS 开发模式已经成为主流模式，但是当我们习以为常的时候，千万别忘记其实 JS 模块化的过程其实是很曲折的，让我们再回顾一下相关的概念：

  1. 什么是模块？

    模块就是可复用的代码片段，封装了具体的实现细节，暴露公开的API，从而方便自己或他人调用。

  2. 为什么要模块化？

    没有模块，我们照样写代码，但是模块化有下面几个好处：

    • 代码抽象：只需要知道某个模块封装的功能、调用时需要传递的参数和返回格式即可，不用关系实现细节。
    • 代码封装：隐藏代码，不会被轻易变更。
    • 代码复用：避免重复劳动。
    • 依赖管理：在代码无需改动的情况下轻松更换依赖。

  1. 在ES5中如何进行模块化？

    ES5并不支持模块，但是有两种方式可以模拟模块：立即执行函数（Immediately Invoked Function Expression, IIFE）和揭示模块设计模式（Revealing Module pattern）。

  3. 模块加载器（module loader）是干什么的？

    就是加载上面定义的模块的，例如 RequireJS 是 AMD 模块加载器，SystemJS 是 System.register 模块加载器，都是在运行时动态加载所需依赖。

  4. 模块打包器（module bundler）是干什么的？

    可以替代模块加载器，是在编译阶段打包所有依赖的，例如 Browserify、Webpack、Parcel 等。
