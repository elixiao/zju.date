数据有效性验证

组件的参数可以随便传，如果传入一个不合法的数据（需要数字却传入字符串），就像闲杂人等进入一个没有安检的大楼，有可能会带来安全隐患。为了避免此类情况发生，最好的方式就是在进入之前统一安检（进行数据有效性验证），剔除危险人物（不符合要求的数据）。

这样，我们的组件既保留了灵活性，又保证了安全性，从此再也不会出现大量的无意义的报错信息了。

处理交互

缺少了交互的网页好比一潭死水，毫无生机，毫无吸引力。看看JSX如何让传统的onclick、onmouseover等事件发挥威力的吧。

添加ref引用

我们在JSX里面有很多元素，在交互的时候往往会要求准确定位到某个元素，如何做到这一点呢？这里提供一个利器：为指定元素添加ref引用。

添加ref引用类似于在CSS里面用ID定位了某个元素，在需要的时候查询器可以迅速找到它。不过值得注意的一点是必须要等dom元素加载之后才可以去获取，否则得到的是null。

状态管理

这部分非常重要，因为react的核心思想是将视图组件化，通过更新组件的state来渲染页面。组件化在component篇章已经介绍过了，那什么是状态管理呢？举个例子：一个人当前所处的状态决定了他接下来将要采取什么动作，如果一个人进入饿的状态，那么他接下来要喝水。在组件当中，用this.state来存储组件的状态，this.setState来改变状态。组件分三种状态：

  1. 挂载
  2. 更新
  3. 卸载

首次装载组件时，按顺序执行

  1. getDefaultProps
  2. getInitialState
  3. componentWillMount
  4. render
  5. componentDidMount

这里看到setState发挥的巨大威力了，通过onChange来监听文本框文字的改变，然后用setState更新状态，状态的更新又触发了render函数，从而更新了视图。

ajax请求

前端开发中ajax的使用是必不可少的，也正是因为ajax的产生促使前端得到强有力的发展，ajax是数据的根源、是页面的灵魂。其实在react中使用ajax非常简单，你可以用任何一种你喜欢的封装，在componentDidMount中获取数据，然后改变state即可。
